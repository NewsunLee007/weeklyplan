# 系统优化方案 (System Optimization Plan - Phase 2)

## 1. 摘要 (Summary)
本次优化旨在进一步提升预发工作指导的规范性及表格填报的智能体验：
1. **预发工作表格化**：将办公室主任填写的“学校预发工作指导”升级为独立的表格页面（与新建计划界面一致），包含日期、工作内容、负责人等列，以便清晰指导全校。
2. **多行录入智能排版**：在所有计划表格（新建、预发、审核）的“工作内容”框失去焦点时，若输入了多行文本，系统会自动编号（1. 2. 3.），并在“负责人”框生成对应的空白编号行以供对齐填写。
3. **增加输入框默认高度**：将所有多行文本框的基础高度增加（从1行提升至3行），提升录入体验。
4. **修复夜间模式显示**：优化预发工作展示区的 CSS，确保在夜间模式下清晰可读。

## 2. 具体修改方案 (Proposed Changes)

### 2.1 预发工作独立页面 (`GuidelineForm.vue`)
- **路由**：在 `frontend/src/router/index.js` 新增 `/guideline/edit` 路由（仅限 `OFFICE_HEAD` 和 `ADMIN`）。
- **页面设计**：新建 `GuidelineForm.vue`。页面布局复刻 `PlanForm.vue`，包含学期、周次选择，以及标准的计划条目表格。
- **数据存储**：前端在保存时，将表格的 `items` 数组使用 `JSON.stringify` 转换为字符串，提交给原有的 `/api/guidelines/save` 接口，存入 `biz_weekly_guideline` 的 `content` 字段。加载时使用 `JSON.parse` 还原。

### 2.2 `PlanForm.vue` 预发展示及交互升级
- **展示优化**：解析获取到的 `guideline.content`，如果是 JSON 数组，则渲染为一个迷你的只读 `<el-table>`（包含日期、星期、工作内容、负责人）。
- **编辑入口**：原“编辑预发工作”按钮不再弹窗，而是跳转至 `/guideline/edit?semester=x&week=y`。
- **夜间模式修复**：更新展示区的 CSS，使用 `var(--color-bg-secondary)` 等适配夜间模式的主题变量。

### 2.3 录入框高度与智能排版 (Smart Input)
- **高度调整**：在 `PlanForm.vue`, `GuidelineForm.vue`, `ReviewDetail.vue`, `ReviewConsolidated.vue` 中，将所有 `type="textarea"` 的 `:autosize="{ minRows: 1, maxRows: 6 }"` 统一修改为 `minRows: 3`。
- **智能编号逻辑**：
  - 绑定 `@blur="onContentBlur(row)"` 事件。
  - **失去焦点时**：检查 `row.content`，按换行符拆分。若超过 1 行，自动为每行首部添加序号（如 `1. `），忽略已包含序号的行。
  - **同步负责人**：根据工作内容的行数，在 `row.responsible` 中生成相等数量的换行，并自动填充序号，方便用户直接在对应序号后输入负责人。

## 3. 假设与决策 (Assumptions & Decisions)
- **数据兼容性**：在 `PlanForm.vue` 解析预发工作时，会使用 `try-catch` 包裹 `JSON.parse`。如果解析失败（如之前测试写入的纯文本），则降级按纯文本显示，保证不报错。
- **排版触发时机**：根据用户选择，智能排版在**失去焦点 (`@blur`)** 时触发，这样不会打断用户正在输入、换行的连贯思维。

## 4. 验证步骤 (Verification Steps)
1. **预发工作**：办公室主任点击“编辑预发工作”，进入独立页面，使用表格形式添加两条工作，保存后返回新建计划页，能看到表格形式的只读预发指导。
2. **夜间模式**：切换到夜间模式，查看预发工作表格或文本，确认对比度正常、文字清晰。
3. **智能排版**：在工作内容框输入三行无序号文本，鼠标点击外部。确认工作内容自动变成带 `1. 2. 3.` 的编号，同时负责人框内自动出现三行 `1. 2. 3.`，并且框的默认高度明显变大。
