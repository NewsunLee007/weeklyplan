# UI及功能细节优化计划

## 1. 现状分析 (Current State Analysis)
根据用户的需求和提供的截图，目前系统存在以下几个体验和视觉上的问题：
1. **左侧导航栏审核中心肩标位置**：当前肩标渲染在文字的正右侧，用户希望将其移至“心”字的右上方。
2. **部门提交计划撤回与再编辑**：目前提交后的计划无法撤回，只能等待审核退回。用户希望部门提交审核后，可以主动撤回，并允许再次编辑。
3. **日期格式多余的 `T00:00:00.000Z`**：在详情页等地方，时间格式未被正确格式化，直接展示了 ISO 后缀，影响阅读体验。需要全局排查并删除。
4. **登录页面简陋**：目前登录页面就是一个白底卡片在渐变背景上，且带有写死的“默认管理员：admin / admin123”提示，不仅不安全，且缺乏设计感。用户提供了一张 Logo 图片链接（`https://p.ipic.vip/lmzked.jpg`），希望加上 Logo 并且背景带有蒙皮动态效果。
5. **整体 UI 设计**：要求提升整体设计感和 UI 布局。

## 2. 拟议修改 (Proposed Changes)

### 2.1 左侧导航栏肩标优化
- **文件**：`frontend/src/layout/MainLayout.vue`
- **修改**：调整 `.review-badge` 的 CSS 样式，将其 `top` 和 `right` 属性微调，使其更精准地落在文字右上角。
  ```css
  .review-badge {
    position: absolute;
    top: -12px; /* 调整高度 */
    right: -20px; /* 调整水平位置 */
  }
  ```

### 2.2 计划撤回与再次编辑
- **后端**：
  - 新增 `POST /api/plans/:id/withdraw` 接口。
  - 允许创建者在计划状态为 `SUBMITTED` 或 `DEPT_APPROVED` 或 `OFFICE_APPROVED`（未被发布前）主动撤回。
  - 撤回后状态重置为 `DRAFT`。
- **前端**：
  - 在 `PlanList.vue` 的操作列和 `PlanDetail.vue` 详情页中，对于已提交但未发布的计划，新增【撤回】按钮。
  - 撤回成功后，由于状态变为 `DRAFT`，原本的【编辑】按钮会自动出现。

### 2.3 修复日期中多余的 `T00:00:00.000Z`
- **文件**：`frontend/src/views/plan/PlanDetail.vue` 及其他涉及显示的地方。
- **修改**：通过 `formatDate` 函数确保所有展示的日期都被截断到 `YYYY-MM-DD`。虽然目前代码里有 `formatDate`，但在某些模板里（如条目的 `item.plan_date`）可能没有使用该函数包裹，导致原生字符串直接展示。

### 2.4 登录页面重构与美化 (动用界面美化思想)
- **文件**：`frontend/src/views/Login.vue`
- **修改**：
  - 删除写死的账号密码提示。
  - 增加动态蒙皮背景：使用深色/磨砂玻璃效果（Glassmorphism）覆盖在一张校园背景图或动态渐变背景上。
  - 顶部居中或卡片上方展示提供的 Logo (`https://p.ipic.vip/lmzked.jpg`)。
  - 优化输入框和按钮的圆角、阴影、间距，使其看起来更像一个现代化的 SaaS 系统。

### 2.5 整体 UI 设计优化
- 对 `PlanList.vue` 和 `MainLayout.vue` 中的卡片边距、字体排版、阴影层级进行统一提升，采用更现代的轻量化投影和圆角（如 `border-radius: 12px`）。

## 3. 验证步骤 (Verification Steps)
1. 检查侧边栏的数字是否位于右上角。
2. 检查登录页面是否展现了带有 Logo 的新设计，且没有密码提示。
3. 提交一份计划后，点击“撤回”，验证计划是否回到草稿状态并可重新编辑。
4. 检查计划详情页中，工作条目的日期是否已经去除了 `T00...Z`。
